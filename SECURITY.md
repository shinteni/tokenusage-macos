# Security policy

## Reporting a vulnerability

Please do not open a public issue for a suspected security vulnerability.

Use the repository's **Security** tab and choose **Report a vulnerability** to
start a private GitHub security advisory. Include the affected version, impact,
reproduction steps, and the minimum redacted evidence needed to validate the
report.

Do not include live API keys, OAuth tokens, session cookies, or unredacted
credential files. If a secret is exposed during investigation, revoke it before
sharing the report.

## Supported version

Security reports are currently accepted for TokenUsage 0.1.0.
